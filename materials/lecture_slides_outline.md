## Slide 11：无线信道的两个核心问题

页面要点：
- 问题1：加性噪声 → 0/1 判决错误 → BER 升高
- 问题2：多径传播 → 不同延迟路径叠加 → ISI
- ISI 使眼图闭合，BER 大幅恶化
- 编码对抗误码：加冗余，接收端能检测并纠正
- 均衡对抗 ISI：逆卷积，补偿信道失真

教师讲稿：
可在白板画出：发送比特 → 编码 → 调制 → 信道（噪声+多径）→ 均衡 → 解调 → 译码。两个问题在接收端先后处理，顺序不能颠倒。

## Slide 12：信道编码的直观思想

页面要点：
- 不直接发送裸信息比特，加入校验冗余
- 合法码字在所有可能序列中只是一小部分
- 若接收序列不是合法码字，说明发生了错误
- 小错误（1 bit 翻转）可被定位并纠正
- 代价：码率 R = k/n < 1，需要更多带宽或时间

教师讲稿：
类比"身份证号码末位校验"：最后一位是冗余信息，用来检测录入错误。Hamming(7,4) 每 4 个信息比特加 3 个校验比特，码率 R = 4/7 约 0.571。

## Slide 13：Hamming(7,4) 参数与能力

页面要点：
- 信息比特 k=4，码字长度 n=7，校验比特 r=3
- 码率：R = 4/7 ≈ 0.571（约57%用于信息）
- 最小距离：d_min = 3，可纠正任意 1 bit 错误
- 系统码：码字前 4 位保留原始信息，后 3 位为校验
- HAMMING_G（4×7）和 HAMMING_H（3×7）已在代码中定义

教师讲稿：
d_min=3 的含义：任意两个合法码字至少有 3 个位置不同。因此 1 个错误后，距离最近的合法码字仍是原始码字，判决正确。

## Slide 14：生成矩阵 G 与编码公式

页面要点：
- G 是 4×7 矩阵，在 GF(2) 二元域上运算
- 编码公式：c = u × G mod 2
- u 是 1×4 行向量，c 是 1×7 码字行向量
- 批量编码：bits.reshape(-1, 4) @ HAMMING_G % 2
- 所有结果必须对 2 取模，保持 0/1 二进制

教师讲稿：
GF(2) 上矩阵乘法与普通乘法相同，但最后必须 mod 2。常见错误是忘记 mod 2，导致码字出现大于 1 的值，测试直接失败。

## Slide 15：编码例子：u = [1, 0, 1, 1]

页面要点：
- 信息比特 u = [1, 0, 1, 1]
- 取 G 中 u 非零对应行（第1、3、4行）求和 mod 2
- 第1行 [1,0,0,0,1,1,0]，第3行 [0,0,1,0,0,1,1]，第4行 [0,0,0,1,1,1,1]
- 按位异或求和：[1,0,1,1,0,1,0]
- 验证：码字前4位 [1,0,1,1] = 原始信息 u ✓

教师讲稿：
鼓励同学们当场手算并用 Python 验证：np.array([1,0,1,1]) @ HAMMING_G % 2。系统码的好处是不需要额外操作就能从码字中读出信息位。

## Slide 16：校验矩阵 H 与伴随式

页面要点：
- H 是 3×7 矩阵，对任意合法码字 c 满足：H × c^T = 0 mod 2
- 伴随式：s = r × H^T mod 2（r 为接收向量）
- s = [0,0,0] 表示未检测到单比特错误
- s ≠ 0 时，s 等于 H 的某列 → 该位置出错
- 实现重点：codewords.reshape(-1,7) @ HAMMING_H.T % 2

教师讲稿：
H 是"合法码字的检测器"。没有错误时过 H 得零向量。有错误时 syndrome 就是错误位的 H 列指纹，通过逐列比较精确定位。

## Slide 17：纠错例子：syndrome 定位错误

页面要点：
- 发送 c = [1,0,1,1,0,1,0]，第3位翻转为 0
- 接收 r = [1,0,0,1,0,1,0]
- 计算 s = r × H^T mod 2 = [0,1,1]
- H 第3列 = [0,1,1]，与 s 完全匹配 → 第3位出错
- 翻转 r[2]：得到 [1,0,1,1,0,1,0]，取前4位 [1,0,1,1] ✓

教师讲稿：
这是 hamming74_decode 的完整实现流程：reshape → syndrome → 逐列比较 → 翻转 → 取前4位。提醒不要修改原始数组，建议 received.copy() 后再操作。

## Slide 18：Hamming 编码增益：BER 对比

页面要点：
- 未编码 BER = p（直接受噪声影响）
- 编码后只有 2 个以上比特同时出错才导致误判
- p=0.01：未编码 BER=0.01，编码后约 0.002（改善 5 倍）
- p=0.05：改善仍明显；p>0.3 时编码增益消失
- 实验生成 results/coding_ber_curve.png 可视化对比

教师讲稿：
BER 曲线是验证编码实现正确性的关键。编码后 BER 曲线应在低翻转概率段明显低于未编码线，高概率段两者趋近。

## Slide 19：选做：卷积码 (2,1,3) 简介

页面要点：
- 输入每 1 bit，输出 2 bit，码率 R = 1/2
- 编码器有记忆：3 个移位寄存器存储历史比特
- 生成多项式：g1 = [1,1,1]，g2 = [1,0,1]
- 每个输出比特 = 当前输入与移位寄存器 XOR 组合
- Viterbi 硬判决译码：按最小汉明距离搜索最优路径

教师讲稿：
卷积码是选做任务，不计入必做分数。与 Hamming 分组码不同，卷积码有编码记忆，纠错能力更强，广泛用于 GSM 等实际系统。

## Slide 20：什么是信道均衡

页面要点：
- 多径：信号经直达、反射、绕射多条路径到达
- 每条路径有不同的延迟和增益，叠加形成 ISI
- 接收信号 = 发送信号 * 信道冲激响应 + 噪声
- 均衡器 = FIR 滤波器，目标：抵消信道卷积失真
- 均衡后信号更接近原始符号，眼图重新张开

教师讲稿：
多径类比：在空旷山谷说话会听到回声，每次回声是一条额外路径。数字通信中每个符号的"回声"叠加到下一个符号，就是 ISI。均衡器的作用是消除这些回声。

## Slide 21：多径信道模型与 ISI 数值例子

页面要点：
- 实验信道：h = [0.9, 0.35, -0.25]（3 径模型）
- y[n] = 0.9·x[n] + 0.35·x[n-1] - 0.25·x[n-2]
- 发送 x = [+1, -1, +1]，计算各采样点
- y[0] = 0.9，y[1] = -0.55，y[2] = 0.30（本应接近+1）
- ISI 使 y[2] 偏离 +1 达 0.70，判决容易出错

教师讲稿：
y[2] 本应接近 +1 但只有 0.30，原因是 x[1]=-1 的残留影响。实验用 np.convolve(symbols, h) 模拟，加上噪声后 BER 会显著升高。

## Slide 22：ZF 迫零均衡原理

页面要点：
- ZF 目标：channel * taps = δ（理想冲激响应）
- 构造卷积矩阵 A（信道响应的 Toeplitz 矩阵）
- 构造目标向量 d（中心为 1、其余为 0）
- 求解：np.linalg.lstsq(A, d)，得到最优均衡器抽头 w
- 实现：estimate_zf_equalizer(channel, num_taps=7)

教师讲稿：
A 矩阵的每一列是信道响应 h 的移位版本。A @ w 等于 channel * w 的卷积，令其等于 δ 就是迫零。lstsq 比矩阵求逆更数值稳定。

## Slide 23：ZF 计算例子：h = [0.9, 0.35, -0.25]

页面要点：
- 已知 h = [0.9, 0.35, -0.25]，num_taps = 5
- 构造 7×5 的 Toeplitz 矩阵 A（3+5-1=7 行）
- 目标 d = [0, 0, 1, 0, 0, 0, 0]（中心在第3位）
- lstsq 求解得 w ≈ [−0.03, −0.16, 1.16, 0.01, 0.05]
- 验证：np.convolve(h, w) ≈ [0, 0, 1, 0, 0] ✓

教师讲稿：
建议当场演示：打印 np.convolve(h, w)，让学生观察中心是否接近1、旁瓣是否接近0。apply_fir_filter 就是把 w 应用到整个接收信号。

## Slide 24：LMS 自适应均衡原理

页面要点：
- LMS = Least Mean Squares，不需要已知信道响应
- 通过训练序列（已知发送符号）迭代学习均衡器抽头
- 初始化 w = [0,...,1,...,0]（中心抽头为1）
- 每步迭代：y=w^T·x，e=d-y，w=w+μ·e·x
- 步长 μ=0.01：控制收敛速度与稳定性的平衡

教师讲稿：
LMS 核心是"试一下 → 看误差 → 往误差减小方向修正"。步长 μ 太大则权重振荡发散，太小则需要更多训练样本才能收敛。

## Slide 25：LMS 训练过程与收敛效果

页面要点：
- 用前 800 个样本（rx[:800] 与 tx[:800]）做训练
- 初始误差 |e| 约 0.4~0.9（权重还未收敛）
- 训练约 200 步后 MSE 快速下降，进入收敛区
- 收敛后 MSE 稳定在 0.02~0.05（效果因 μ 而异）
- 训练完成的 w 用于均衡整段接收信号

教师讲稿：
提醒学生注意输入向量 x 的方向和长度：每步取 rx[n:n+num_taps] 作为输入向量，与 tx[n] 对应。方向对齐错误会导致 MSE 不下降。

## Slide 26：总结：编码与均衡在无线链路中的位置

页面要点：
- 发送：信息比特 → Hamming(7,4) 编码 → BPSK 调制 → 发射
- 信道：多径卷积 + 加性噪声（两问题同时存在）
- 接收：ZF/LMS 均衡 → BPSK 解调 → Hamming 译码 → 恢复比特
- 两者协同：均衡改善判决质量，编码纠正剩余误码
- Q&A：欢迎提问环境配置 / 原理 / 代码实现

教师讲稿：
再次强调均衡在解调之前，译码在解调之后。如果均衡效果差，编码也无法弥补过多误码。实验中可对比有无均衡时的最终 BER 差异，加深理解。
